Journald task executor

Manages systemd journal configuration files and drop-in configurations.

Â§Examples

Â§Configure journald storage and rotation

This example configures journald to use persistent storage and sets size limits.

YAML Format:

- type: journald
  description: "Configure systemd journal settings"
  config:
    Storage: persistent
    SystemMaxUse: 100M
    SystemKeepFree: 500M
    SystemMaxFileSize: 10M
    MaxRetentionSec: 1week
  state: present

JSON Format:

{
  "type": "journald",
  "description": "Configure systemd journal settings",
  "config": {
    "Storage": "persistent",
    "SystemMaxUse": "100M",
    "SystemKeepFree": "500M",
    "SystemMaxFileSize": "10M",
    "MaxRetentionSec": "1week"
  },
  "state": "present"
}

TOML Format:

[[tasks]]
type = "journald"
description = "Configure systemd journal settings"
[tasks.config]
Storage = "persistent"
SystemMaxUse = "100M"
SystemKeepFree = "500M"
SystemMaxFileSize = "10M"
MaxRetentionSec = "1week"
state = "present"

Â§Create a drop-in configuration

This example creates a drop-in configuration file for journald.

YAML Format:

- type: journald
  description: "Configure journald forwarding"
  name: forwarding
  config:
    ForwardToSyslog: "yes"
    ForwardToKMsg: "no"
    ForwardToConsole: "no"
    ForwardToWall: "no"
  state: present

Â§Remove journald configuration

This example removes a journald drop-in configuration.

YAML Format:

- type: journald
  description: "Remove custom journald config"
  name: custom-config
  state: absent
